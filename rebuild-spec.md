# Full-Stack Rebuild Spec

**Slug:** `fullstack-rebuild`
**Date:** 2026-05-13
**Scope:** Complete rewrite — plain JS/CJS/React 18 → TypeScript 5 ESM stack per `prd.md`.

---

## §1 Summary

Replace every source file across backend, frontend, DB, and infra with the target PRD stack. The entity is renamed `items → products` with an extended schema (price, stock, image_path). Frontend is extended beyond the PRD's read-only List+Detail to include Create+Edit forms (per user decision). All three layers are replaced simultaneously — no incremental migration. The control script is renamed `deploy.sh → manage.sh` and `docker-compose.dev.yml` moves to the repo root.

## §3 Architecture Patterns Applied

- **Standard layered backend**: router → controller → service → db/client. No advanced patterns.
- **StorageProvider interface**: `LocalDiskStorage` default; S3 swap requires only new implementation.
- **DomainEventBus interface**: `NoOpEventBus` wired via constructor injection; Kafka swap requires only new implementation + re-wire in `index.ts`.
- **Observability — Pino only**: structured JSON logs with `requestId` per request (OTEL `traceId` slot). No OTEL SDK, no collector, no Grafana.
- **TanStack Query**: all server state; no manual `useEffect` fetching.
- **TanStack Router v1 file-based**: route tree auto-generated by Vite plugin.

No WebSockets, no distributed events infra, no AI/LLM, no RSC/Next.js, no edge runtime.

---

## §4 Design Decisions

1. **Complete rewrite**: study project, no prod data. Cleaner than incremental migration.
2. **items → products**: entity fully renamed. Fresh `V001__create_products.sql` replaces `V1__init.sql`. Old migration deleted.
3. **Full CRUD frontend**: user decision — extends PRD's List+Detail to also include Create + Edit routes. Mutations via TanStack Query `useMutation`, with query cache invalidation.
4. **Constructor injection for DomainEventBus**: `ProductsService(db, eventBus)` wired in `index.ts`. Swap to Kafka: implement `KafkaEventBus`, change one line in `index.ts`. No other changes.
5. **Module singletons for db**: `db/client.ts` exports `db` singleton (Drizzle instance). Service takes it as constructor arg.
6. **tsx for dev**: replaces nodemon. No compilation step in dev. `tsc` for production build.
7. **Drizzle Kit migrations output** → `../../db/mysql/migrations/` (relative to `backend/nodejs/`), keeping migrations co-located with DB assets.
8. **Docker seed**: seed.sql mounted into MySQL initdb.d as `Z001__seed.sql` (sorts after `V001__`). Idempotent — inserts only if table empty.
9. **Vite proxy in native dev**: `VITE_API_BASE_URL=http://localhost:3000/api/v1`. Nginx proxy in Docker mode.
10. **deploy.sh → manage.sh**: renamed, updated for new structure (PIDs in `.pids/`, `docker-compose.dev.yml` at root).

## §5 API Contract

Base path: `/api/v1`. All responses `Content-Type: application/json`.

| Method | Path | Auth? | Request | Response shape | Status codes |
|--------|------|-------|---------|----------------|--------------|
| GET | `/api/v1/products` | No | `?page=1&limit=20` | `{data: Product[], meta: {total, page, limit}}` | 200 |
| GET | `/api/v1/products/:id` | No | — | `{data: Product}` | 200, 404 |
| POST | `/api/v1/products` | No | JSON body | `{data: Product}` | 201, 400 |
| PUT | `/api/v1/products/:id` | No | JSON body (partial) | `{data: Product}` | 200, 400, 404 |
| DELETE | `/api/v1/products/:id` | No | — | 204 No Content | 204, 404 |
| POST | `/api/v1/products/:id/image` | No | `multipart/form-data` field `file` | `{data: Product}` | 200, 400, 404 |
| GET | `/health` | No | — | `{status, uptime, db}` | 200 |

**Product shape:**
```json
{
  "id": 1,
  "name": "Running Shoes",
  "description": "Lightweight running shoes",
  "price": "89.99",
  "stock": 50,
  "image_path": null,
  "created_at": "2026-05-13T10:00:00.000Z",
  "updated_at": "2026-05-13T10:00:00.000Z"
}
```

Note: `price` is a string in JSON output (DECIMAL serialized by mysql2 as string).

**POST /api/v1/products — request:**
```json
{ "name": "Running Shoes", "description": "Lightweight running shoes", "price": 89.99, "stock": 50 }
```
`description` optional. `stock` optional (default 0). `price` required, number ≥ 0.

**Error shape (all errors):**
```json
{ "error": { "code": "NOT_FOUND", "message": "Product not found" } }
```

**Error codes:** `VALIDATION_ERROR` (400), `NOT_FOUND` (404), `FILE_UPLOAD_ERROR` (400), `INTERNAL_ERROR` (500).

**List response:**
```json
{
  "data": [...],
  "meta": { "total": 42, "page": 1, "limit": 20 }
}
```

**Health response:**
```json
{ "status": "ok", "uptime": 123.45, "db": "ok" }
```
`"db": "error"` (not 500) if DB unreachable.

Pagination defaults: `page=1`, `limit=20`, max `limit=100`.

### Migration file

**File:** `db/mysql/migrations/V001__create_products.sql`
**Delete:** `db/mysql/migrations/V1__init.sql`

```sql
CREATE TABLE products (
  id          INT UNSIGNED    NOT NULL AUTO_INCREMENT,
  name        VARCHAR(255)    NOT NULL,
  description TEXT,
  price       DECIMAL(10,2)   NOT NULL,
  stock       INT UNSIGNED    NOT NULL DEFAULT 0,
  image_path  VARCHAR(500),
  created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME                 DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

Down-migration (for reference — not wired into tooling):
```sql
DROP TABLE IF EXISTS products;
```

### Drizzle schema (mirrors migration)

**File:** `backend/nodejs/src/db/schema/products.ts`

```typescript
import { mysqlTable, int, varchar, text, decimal, datetime } from 'drizzle-orm/mysql-core'
import { sql } from 'drizzle-orm'

export const products = mysqlTable('products', {
  id:          int('id').unsigned().primaryKey().autoincrement(),
  name:        varchar('name', { length: 255 }).notNull(),
  description: text('description'),
  price:       decimal('price', { precision: 10, scale: 2 }).notNull(),
  stock:       int('stock').unsigned().notNull().default(0),
  image_path:  varchar('image_path', { length: 500 }),
  created_at:  datetime('created_at').notNull().default(sql`CURRENT_TIMESTAMP`),

})

export type Product = typeof products.$inferSelect
export type NewProduct = typeof products.$inferInsert
```

**10. `backend/nodejs/src/storage/local.storage.ts`** — new

```typescript
import { promises as fs } from 'node:fs'
import path from 'node:path'
import { config } from '../config.js'
import type { StorageProvider } from './storage.interface.js'

export class LocalDiskStorage implements StorageProvider {
  private readonly dir = path.resolve(config.UPLOAD_DIR)

  async save(file: Buffer, filename: string, _mimetype: string): Promise<string> {
    await fs.mkdir(this.dir, { recursive: true })
    const dest = path.join(this.dir, filename)
    await fs.writeFile(dest, file)
    return `/uploads/${filename}`
  }

  async delete(filePath: string): Promise<void> {
    const full = path.join(this.dir, path.basename(filePath))
    await fs.unlink(full).catch(() => undefined)
  }

  getUrl(filePath: string): string {
    return filePath
  }
}
```

```typescript
import type { ZodSchema } from 'zod'

export const validate = (schema: ZodSchema, target: 'body' | 'query' = 'body'): RequestHandler =>
  (req, res, next) => {
    const result = schema.safeParse(req[target])
    if (!result.success) {
      res.status(400).json({
        error: { code: 'VALIDATION_ERROR', message: result.error.message },
      })
      return
    }
    req[target] = result.data
    next()
  }
```

```typescript
import type { RequestHandler } from 'express'
import { logger } from '../logger.js'

export const requestLogger: RequestHandler = (req, res, next) => {
  const requestId = randomUUID()
  res.locals['requestId'] = requestId
  res.setHeader('X-Request-Id', requestId)

  const start = Date.now()
  res.on('finish', () => {
    logger.info({
      requestId,
      method: req.method,
      url: req.originalUrl,
      statusCode: res.statusCode,
      responseTimeMs: Date.now() - start,
    }, 'request completed')
  })
  next()
}
```

**18. `backend/nodejs/src/products/products.router.ts`** — replaces `src/routes/items.js`

```typescript
import { Router } from 'express'
import multer from 'multer'
import { validate } from '../middleware/validate.js'
import { createProductSchema, updateProductSchema } from './products.schema.js'
import type { ProductsController } from './products.controller.js'
import type { StorageProvider } from '../storage/storage.interface.js'
import { config } from '../config.js'

export const createProductsRouter = (
  controller: ProductsController,
  storage: StorageProvider,
) => {
  const router = Router()
  const upload = multer({
    storage: multer.memoryStorage(),
    limits: { fileSize: config.MAX_FILE_SIZE_MB * 1024 * 1024 },
    fileFilter: (_req, file, cb) => {
      const allowed = ['image/jpeg', 'image/png', 'image/gif', 'image/webp']
      cb(null, allowed.includes(file.mimetype))
    },
  })

  router.get('/',        controller.list)
  router.get('/:id',     controller.get)
  router.post('/',       validate(createProductSchema),  controller.create)
  router.put('/:id',    validate(updateProductSchema),  controller.update)
  router.delete('/:id',  controller.remove)
  router.post('/:id/image', upload.single('file'), controller.uploadImage)

  return router
}
```

```typescript
export const createApp = (
  db: Db,
  controller: ProductsController,
  storage: StorageProvider,

  app.use(cors())

  app.use(errorHandler)
  return app
```

```typescript
const BASE = import.meta.env['VITE_API_BASE_URL'] as string

export interface Product {
  id: number
  name: string
  description: string | null
  price: string
  stock: number
  image_path: string | null
  created_at: string
  updated_at: string
}

export interface ProductListMeta { total: number; page: number; limit: number }
export interface ProductListResponse { data: Product[]; meta: ProductListMeta }
export interface ProductResponse    { data: Product }
export interface CreateProductInput { name: string; description?: string; price: number; stock?: number }
export interface UpdateProductInput { name?: string; description?: string; price?: number; stock?: number }

const handleResponse = async <T>(res: Response): Promise<T> => {
  if (!res.ok) {
    const body = await res.json().catch(() => ({}))
    throw new Error(body?.error?.message ?? `HTTP ${res.status}`)
  }
  return res.json() as Promise<T>
}

export const fetchProducts = (page = 1, limit = 20) =>
  fetch(`${BASE}/products?page=${page}&limit=${limit}`)
    .then(r => handleResponse<ProductListResponse>(r))

export const fetchProduct = (id: number) =>
  fetch(`${BASE}/products/${id}`)
    .then(r => handleResponse<ProductResponse>(r))

export const createProduct = (input: CreateProductInput) =>
  fetch(`${BASE}/products`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(input),
  }).then(r => handleResponse<ProductResponse>(r))

export const updateProduct = (id: number, input: UpdateProductInput) =>
  fetch(`${BASE}/products/${id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(input),
  }).then(r => handleResponse<ProductResponse>(r))

export const deleteProduct = (id: number) =>
  fetch(`${BASE}/products/${id}`, { method: 'DELETE' })
    .then(r => { if (!r.ok && r.status !== 204) throw new Error(`HTTP ${r.status}`) })

export const uploadProductImage = (id: number, file: File) => {
  const fd = new FormData()
  fd.append('file', file)
  return fetch(`${BASE}/products/${id}/image`, { method: 'POST', body: fd })
    .then(r => handleResponse<ProductResponse>(r))
}
```

## §12 Validation & Error Handling

**Input validation:**
- `validate(schema)` middleware in `src/middleware/validate.ts`
- Applied per-route in `products.router.ts` before controller: POST uses `createProductSchema`, PUT uses `updateProductSchema`
- Query params validated in controller via `paginationSchema.parse(req.query)`
- Multer file filter rejects non-image MIME types (400 returned by error handler)

**Error taxonomy:**

| Code | HTTP | Trigger |
|------|------|---------|
| `VALIDATION_ERROR` | 400 | Zod schema failure, multer MIME reject |
| `NOT_FOUND` | 404 | Product not found (service throws AppError) |
| `FILE_UPLOAD_ERROR` | 400 | No file in multipart request |
| `INTERNAL_ERROR` | 500 | Unhandled exception |

**AppError:** `src/errors.ts` — `class AppError extends Error { status, code }`. Service throws, global handler reads.

**Global handler:** `src/middleware/errorHandler.ts` — last middleware in `app.ts`. Logs `error` level with full stack. In dev mode: includes `err.message` in response. In prod: generic message.

**Express 5:** async errors auto-forwarded to error handler. Explicit try/catch in controller ensures `AppError` status codes are preserved.

## §13 Auth & Security Notes

- No auth (out of scope per PRD).
- **MIME allowlist** in multer fileFilter: `['image/jpeg', 'image/png', 'image/gif', 'image/webp']`. Non-matching files rejected before reaching controller.
- **File size limit**: `MAX_FILE_SIZE_MB * 1024 * 1024` bytes enforced by multer `limits.fileSize`.
- **Parameterized queries**: Drizzle ORM generates prepared statements. No raw SQL string interpolation.
- **Env secrets**: `DATABASE_URL` contains credentials — in `.gitignore`, not logged (Pino base fields contain no credentials), not in client bundle.
- **Client bundle**: no server env vars exposed. Frontend only has `VITE_API_BASE_URL` (public API endpoint — safe).
- **CORS**: `cors()` middleware with default config (all origins). Tighten via `cors({ origin: ... })` when deploying.

## §20 Risk Summary & Open Questions

### Risk table

| Risk | Breaking? | Data-loss? | Perf impact? | Severity | Mitigation |
|------|-----------|-----------|-------------|----------|------------|
| `items` table dropped, `products` created | Yes (API) | Yes (seed data only — no real data) | None | Low | Fresh DB via `manage.sh reset-db`. Seed is idempotent. |
| `@types/express` v5 types may have gaps | No | No | None | Low | `skipLibCheck: true` in tsconfig. Pin to known-good version. |
| TanStack Router plugin package name | No | No | None | Low | Verify `@tanstack/router-plugin` vs `@tanstack/router-vite-plugin` at `npm install` time — names have changed across minor versions. |
| Drizzle `out` pointing outside package dir | No | No | None | Low | `../../db/mysql/migrations` — works, but `drizzle-kit generate` must be run from `backend/nodejs/`. Document in manage.sh. |
| `pino-pretty` as devDependency in Docker | No | No | None | Low | In Docker build, `npm ci --omit=dev` skips pino-pretty. Prod uses plain JSON (no transport). Config guards on `NODE_ENV`. |
| mysql2 returning `price` as string | No | No | None | Low | Documented in §5. Frontend types `price` as `string`. Display with `$` prefix. |

### Open questions

1. **Uploads in Docker**: `uploads/` volume is named in compose. If backend is scaled (future), uploads need shared storage (S3). Not a current concern but document the `StorageProvider` swap path.
2. **`drizzle-kit migrate` vs Docker initdb.d**: Two migration paths exist. Ensure they stay in sync — initdb.d runs raw SQL, Drizzle Kit tracks migration state in `__drizzle_migrations` table. In Docker mode, initdb.d applies SQL directly (no Drizzle state tracking). Acceptable for study project; for production, choose one path.
3. **TanStack Router `routeTree.gen.ts`**: Must be gitignored or committed? Generated file — should be committed (same as Prisma client) so CI builds don't require a dev step. Add to `.gitignore` exclusion: `!src/routeTree.gen.ts`.
